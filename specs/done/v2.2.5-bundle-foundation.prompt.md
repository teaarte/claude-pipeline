### Item 8 — Structured gate-answer protocol (closes Q57)

**Fix:**

1. New type in `mcp/src/driver/types/plugin.ts`:


   ```typescript
   { type: "user-answer", decision: "accept" | "reject", message?: string }
   ```

   ```typescript
   function parseDecision(input: UserAnswer): { status: "approved" | "rejected"; feedback: string | null } {
   ```

### Item 9 — Classifier-agent + `pickFromCandidates` primitive (closes Q41 / Q44 / Q45 / Q46 / Q58 / Q61)

**Symptom:** Multiple decisions (security_needed, refs-to-load) and one hook (anti-pattern-grep) use English-only regex to classify free-text task descriptions or extract overlapping tokens from prose rules. Non-English tasks → false-negatives. Slug from non-Latin task → useless hex (e.g., `t-2026-05-17-c797`). Category 3 problem — needs LLM picking from candidate set. Architectural fix: single classifier-agent in `context` phase emits structured JSON consumed by all downstream decisions, via existing shuttle infrastructure (no daemon required).

**Fix:**

1. New file `mcp/src/lib/pick-from-candidates.ts` (~55 lines):
   ```typescript
   export interface SpawnQuery {
     query: (args: { prompt: string; model?: string; max_tokens?: number; output_format?: "json-array" }) => Promise<string>;
   }
   
   export async function pickFromCandidates<T extends string>(args: {
     query: SpawnQuery["query"] | undefined;
     task: string;
     candidates: T[];
     cap: number;
     context_hint?: string;
   }): Promise<T[]> {
     if (!args.query || args.candidates.length === 0) return [];
     const prompt = buildPrompt(args);
     try {
       const raw = await args.query({ prompt, model: "haiku", max_tokens: 200, output_format: "json-array" });
       return parsePicked(raw, args.candidates).slice(0, args.cap);
   ```
