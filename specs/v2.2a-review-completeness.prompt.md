### Item 1 — Q43: `impl_iters` overcount fix

**Severity:** 🟢 LOW. Mechanical mis-derivation.

**Symptom:** Real-task `t-...-addauthtokendecodert` metrics row had `impl_iters: 2`. Reality: implementation review loop was a single pass (logic-reviewer APPROVE first try in implementation phase). The `2` came from `max(reviewer_verdicts.iteration WHERE phase=implementation) = 2` — but iter=2 means "2nd time this agent ran on this task overall" (logic-reviewer iter=1 in planning, iter=2 in implementation). Iteration counter is global-per-agent, not per-phase.

**Fix:**

1. Open `mcp/src/tools/finish.ts`. Find `extractMetricsRow` (or whatever Q22 named the derivation logic for `impl_iters`/`plan_iters`).
2. Change derivation from `max(iteration)` to `count` of verdicts filtered by phase:
   ```ts
   const implVerdicts = state.reviewer_verdicts.filter(v => v.phase === 'implementation');
   const impl_iters = implVerdicts.length;  // or implVerdicts.length || 0
   ```
   Same change for `plan_iters` (filter phase='planning').
3. Existing fallback path (legacy `phases.<x>.iterations`) should be removed — those fields were deprecated in Q31 anyway. If they're still consulted, drop the fallback.

**Tests** (`mcp/test/tools/finish.test.ts` — extend):
- Cross-phase same-agent fixture: logic-reviewer iter=1 in planning + iter=2 in implementation → `impl_iters: 1, plan_iters: 1`.
- Single-phase fixture: logic-reviewer iter=1 + iter=2 both in implementation → `impl_iters: 2`.
- Empty phase: no reviewer_verdicts for impl → `impl_iters: 0`.

**Acceptance:** tests pass; mental model "impl_iters = number of distinct review passes in implementation phase" matches the field name.

**Commit subject:** `fix(finish): Q43 — derive impl_iters/plan_iters by count, not max(iteration)`

#### Q41.3 — `refs-to-load` rewrite

In `mcp/src/driver/builtin/decisions/refs-to-load.ts`:
```ts
export const refsToLoadDecision: DecisionPlugin<string[]> = {
  name: "refs_to_load",
  async decide(state, ctx) {
    const cached = state.decisions.refs_to_load;
    if (Array.isArray(cached)) return cached;

    // 1. Load metadata-only from all refs (parse frontmatter)
    const refsMetadata = await loadRefsMetadata("agents/references/");
    // refsMetadata: Array<{filename, tags, stack_signals, summary, when_to_load, agent_hints}>

    // 2. Build LLM prompt
    const prompt = buildSelectionPrompt({
      task: state.task,
      stack: state.stack,
      active_agents: ctx.active_agents,
      available_refs: refsMetadata,
      cap: 5,
    });

    // 3. Single LLM classification call via SpawnProvider
    try {
      const result = await ctx.spawn_provider.query({
        prompt,
        model: "haiku",  // cheap classification, opus when haiku unavailable
        max_tokens: 200,
        output_format: "json-array",
      });
      const picked = parsePickedRefs(result);
      return picked.slice(0, 5);
    } catch (err) {
      // Fallback: empty list. Pipeline continues without refs.
      // Audit emits error_class: "ref-classification-failed"
      return [];
    }
  },
};
```
