   - The AgentPlugin's `template_path` file content (read via `fs.readFile`)
   - Context for the task (driver_state_id, task description, any other context the agent needs)
   - Optionally a header like `# You are the <agent name> agent` so the general-purpose agent knows what role it's playing
3. Audit other code paths that might construct `claude_code_task` — `mcp/src/driver/builtin/steps/index.ts`, anywhere `subagent_type` appears as a literal string. None should use `req.agent` or similar.

**Tests** (`mcp/test/driver/builtin/spawn/shuttle-provider.test.ts`):
- `spawn(req)` returns `claude_code_task.subagent_type === "general-purpose"` regardless of `req.agent`. Test with at least 3 agent names: `planner`, `code-analyzer`, `logic-reviewer`.

### Item 2 — Q7: slug sanitizer in `pipeline_init`

**Severity:** 🔴 HIGH. Currently blocks `/done` for any task whose title contains characters other than `[a-z0-9]`.

**Symptom:** `pipeline_init` generated `task_id: "t-2026-05-13-gateway-ui-gateway-orval-tanstaack-query"`. Schema requires `^t-\d{4}-\d{2}-\d{2}-[a-z0-9]{4,}$` — alphanumeric only after the date. The hyphens fail validation. Bonus: "tanstaack" typo from user input was preserved.

**Fix:**

1. Open `mcp/src/tools/init.ts`. Find the slugifier (likely takes the task description and produces the slug part of `task_id`).
2. Rewrite the slugifier to:
   - Lowercase
   - Replace any non-`[a-z0-9]` character with empty string (don't replace with hyphens — they're not allowed per schema)
   - Truncate to a reasonable max length (suggest 20 chars to leave room — total task_id ≤ ~40 chars feels right; final pattern: `t-YYYY-MM-DD-` (13 chars) + slug ≤ 20 chars = total ≤ 33).
   - Minimum length 4 chars (matches schema `{4,}`). If after sanitization the slug is shorter, pad with random hex from `crypto.randomBytes`.

3. Add a unit test (`mcp/test/tools/init.test.ts`) with cases:
   - `"rename foo to bar"` → slug like `"renamefoobar"`
   - `"feat: Add user-settings page (Phase 0.5)"` → slug like `"featadduserseting"` (truncated to 20)
   - `"кириллица в названии"` → slug must be valid (Cyrillic non-matchable → strip → pad with random hex if too short)
   - `""` (empty input) → slug is valid random hex of length ≥ 4
   - `"a"` (single char) → slug pads to length ≥ 4
   - `"--- ??? !!!"` (all non-alphanumeric) → slug is valid random hex

4. Verify the resulting `task_id` always matches `/^t-\d{4}-\d{2}-\d{2}-[a-z0-9]{4,}$/` — add this regex assertion to the test.

**Acceptance:**
1. Tests above pass.
2. `pipeline_validate` returns `ok: true` immediately after `pipeline_init` for any task description (no `INV_SCHEMA_STATE` on task_id).
3. Roadmap Q7 row marked fixed with commit SHA.

**Commit subject:** `fix(init): Q7 — sanitize task_id slug to match schema pattern`
